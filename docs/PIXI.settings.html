@pixi/settings

PIXI.settings

static namespace settings

User's customizable globals for overriding the default PIXI settings, such as a renderer's default resolution, framerate, float precision, etc.

Public Properties

ANISOTROPIC_LEVEL

static ANISOTROPIC_LEVEL: number

Default anisotropic filtering level of textures. Usually from 0 to 16

CAN_UPLOAD_SAME_BUFFER

static CAN_UPLOAD_SAME_BUFFER: boolean

Can we upload the same buffer in a single frame?

CREATE_IMAGE_BITMAP

static CREATE_IMAGE_BITMAP: boolean

Enables bitmap creation before image load. This feature is experimental.

FAIL_IF_MAJOR_PERFORMANCE_CAVEAT

static FAIL_IF_MAJOR_PERFORMANCE_CAVEAT: boolean

Should the failIfMajorPerformanceCaveat flag be enabled as a context option used in the isWebGLSupported function. For most scenarios this should be left as true, as otherwise the user may have a poor experience. However, it can be useful to disable under certain scenarios, such as headless unit tests.

FILTER_RESOLUTION

static FILTER_RESOLUTION: number

Default filter resolution.

GC_MAX_CHECK_COUNT

static GC_MAX_CHECK_COUNT: number

Default Garbage Collection maximum check count.

GC_MAX_IDLE

static GC_MAX_IDLE: number

Default Garbage Collection max idle.

GC_MODE

static GC_MODE: PIXI.GC_MODES

Default Garbage Collection mode.

MESH_CANVAS_PADDING

static MESH_CANVAS_PADDING: number

Default canvasPadding for canvas-based Mesh rendering.

MIPMAP_TEXTURES

static MIPMAP_TEXTURES: PIXI.MIPMAP_MODES

If set to true WebGL will attempt make textures mimpaped by default. Mipmapping will only succeed if the base texture uploaded has power of two dimensions.

PRECISION_FRAGMENT

static PRECISION_FRAGMENT: PIXI.PRECISION

Default specify float precision in fragment shader. iOS is best set at highp due to https://github.com/pixijs/pixi.js/issues/3742

PRECISION_VERTEX

static PRECISION_VERTEX: PIXI.PRECISION.html

Default specify float precision in vertex shader.

PREFER_ENV

static PREFER_ENV: number

The maximum support for using WebGL. If a device does not support WebGL version, for instance WebGL 2, it will still attempt to fallback support to WebGL 1. If you want to explicitly remove feature support to target a more stable baseline, prefer a lower environment.

Due to bug in chromium we disable webgl2 by default for all non-apple mobile devices.

RENDER_OPTIONS

static RENDER_OPTIONS: object

The default render options if none are supplied to PIXI.Renderer.html or PIXI.CanvasRenderer.html.

RESOLUTION

static RESOLUTION: number

Default resolution / device pixel ratio of the renderer.

RETINA_PREFIX

static RETINA_PREFIX: RegExp

The prefix that denotes a URL is for a retina asset.

ROUND_PIXELS

static ROUND_PIXELS: boolean

If true PixiJS will Math.floor() x/y values when rendering, stopping pixel interpolation. Advantages can include sharper image quality (like text) and faster rendering on canvas. The main disadvantage is movement of objects may appear less smooth.

SCALE_MODE

static SCALE_MODE: PIXI.SCALE_MODES

Default scale mode for textures.

SORTABLE_CHILDREN

static SORTABLE_CHILDREN: boolean

Sets the default value for the container property 'sortableChildren'. If set to true, the container will sort its children by zIndex value when updateTransform() is called, or manually if sortChildren() is called.

This actually changes the order of elements in the array, so should be treated as a basic solution that is not performant compared to other solutions, such as @link https://github.com/pixijs/pixi-display

Also be aware of that this may not work nicely with the addChildAt() function, as the zIndex sorting may cause the child to automatically sorted to another position.

SPRITE_BATCH_SIZE

static SPRITE_BATCH_SIZE: number

The default sprite batch size.

The default aims to balance desktop and mobile devices.

SPRITE_MAX_TEXTURES

static SPRITE_MAX_TEXTURES: number

The maximum textures that this device supports.

STRICT_TEXTURE_CACHE

static STRICT_TEXTURE_CACHE: boolean

If set to true, only Textures and BaseTexture objects stored in the caches (TextureCache and BaseTextureCache) can be used when calling Texture.from or BaseTexture.from. Otherwise, these from calls throw an exception. Using this property can be useful if you want to enforce preloading all assets with PIXI.Loader.html.

TARGET_FPMS

static TARGET_FPMS: number

Target frames per millisecond.

TRANSFORM_MODE

static TRANSFORM_MODE: PIXI.html#TRANSFORM_MODE

Default transform type.

UPLOADS_PER_FRAME

static UPLOADS_PER_FRAME: number

Default number of uploads per frame using prepare plugin.

WRAP_MODE

static WRAP_MODE: PIXI.WRAP_MODES

Default wrap modes that are supported by pixi.
