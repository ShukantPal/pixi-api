@pixi/core

PIXI.Renderer

class Renderer extends PIXI.AbstractRenderer

The Renderer draws the scene and all its content onto a WebGL enabled canvas.

This renderer should be used for browsers that support WebGL.

This renderer works by automatically managing WebGLBatchesm, so no need for Sprite Batches or Sprite Clouds. Don't forget to add the view to your DOM or you will not see anything!

Renderer is composed of systems that manage specific tasks. The following systems are added by default whenever you create a renderer:

System Description
PIXI.BatchSystem.html This manages object renderers that defer rendering until a flush.
PIXI.ContextSystem.html This manages the WebGL context and extensions.
PIXI.FilterSystem.html This manages the filtering pipeline for post-processing effects.
PIXI.FramebufferSystem.html This manages framebuffers, which are used for offscreen rendering.
PIXI.GeometrySystem.html This manages geometries & buffers, which are used to draw object meshes.
PIXI.MaskSystem.html This manages masking operations.
PIXI.ProjectionSystem.html This manages the projectionMatrix, used by shaders to get NDC coordinates.
PIXI.RenderTextureSystem.html This manages render-textures, which are an abstraction over framebuffers.
PIXI.ScissorSystem.html This handles scissor masking, and is used internally by MaskSystem
PIXI.ShaderSystem.html This manages shaders, programs that run on the GPU to calculate 'em pixels.
PIXI.StateSystem.html This manages the WebGL state variables like blend mode, depth testing, etc.
PIXI.StencilSystem.html This handles stencil masking, and is used internally by MaskSystem
PIXI.TextureSystem.html This manages textures and their resources on the GPU.
PIXI.TextureGCSystem.html This will automatically remove textures from the GPU if they are not used.

The breadth of the API surface provided by the renderer is contained within these systems.

clearBeforeRender

clearBeforeRender: boolean

This sets if the CanvasRenderer will clear the canvas or not before the new render pass. If the scene is NOT transparent PixiJS will use a canvas sized fillRect operation every frame to set the canvas background color. If the scene is transparent PixiJS will use clearRect to clear the canvas every frame. Disable this by setting this to false. For example, if your game has a canvas filling background image you often don't need this set.
